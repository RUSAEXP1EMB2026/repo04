## 1. システム概要

### 1.1 目的

Nature Remo 3 を用いて室内の温度・湿度・照度を 5 分おきに取得し、不快指数および照度に基づいてエアコンを自動制御する。あわせて、エアコンの稼働状況や室内環境を Twitter に投稿する。

### 1.2 不快指数の計算式

```
DI = 0.81 × 温度[℃] + 0.01 × 湿度[%] × (0.99 × 温度[℃] − 14.3) + 46.3
```

この値はスプレッドシート内の数式で計算し、センサデータと合わせて記録する。

### 1.3 システム構成

```
[Nature Remo 3]
    |
    | Nature API (HTTPS)
    v
[Google Apps Script]
    |─────────────────────────────────┐
    |                                 |
    | Sheets API (読み書き)            | Twitter API (投稿)
    v                                 v
[Google スプレッドシート]          [Twitter]
  ・センサデータ記録                 ・エアコン操作時
  ・不快指数計算 (関数)              ・毎時00分の定期投稿
  ・ユーザー設定
    |
    | Nature API (エアコン操作)
    v
[エアコン]
```

## 2. アーキテクチャ設計

### 2.1 実行モデル

Google Apps Script のトリガーを 2 種類設定する。

| トリガー種別 | 間隔 | 呼び出す関数 | 役割 |
|------------|------|------------|------|
| 時間主導型（分ベース） | 5 分ごと | `mainProcess()` | センサ取得・エアコン制御 |
| 時間主導型（時ベース） | 毎時 00 分 | `postHourlyTweet()` | 定期 Twitter 投稿 |

### 2.2 スクリプトファイル構成

| ファイル名 | 役割 |
|-----------|------|
| `Main.gs` | エントリーポイント。各モジュールの呼び出し |
| `SensorModule.gs` | Nature API からの温度・湿度・照度取得 |
| `SheetModule.gs` | スプレッドシートへの設定読み取り・データ書き込み |
| `AirconModule.gs` | エアコン状態管理・制御判断・Nature API 操作 |
| `TwitterModule.gs` | Twitter API への投稿処理 |
| `Config.gs` | 定数・デフォルト値の定義 |
| `Utils.gs` | 共通ユーティリティ関数 |

## 3. モジュール設計

### 3.1 SensorModule（センサデータ取得モジュール）

#### 責務

Nature API（`GET /1/devices`）を呼び出し、室内の温度・湿度・照度を取得する。

#### 主要関数

```javascript
/**
 * Nature Remo 3 からセンサデータを取得する
 * @returns {{ temperature: number, humidity: number, illuminance: number }}
 */
function fetchSensorData()
```

#### 取得フィールドの対応

| センサ種別 | API フィールド | 単位 |
|-----------|--------------|------|
| 温度 | `newest_events.te.val` | ℃ |
| 湿度 | `newest_events.hu.val` | % |
| 照度 | `newest_events.il.val` | lux |

---

### 3.2 SheetModule（スプレッドシート管理モジュール）

#### 責務

- ユーザーが設定シートに入力した設定値を読み込む
- センサデータ・不快指数・制御ログをデータシートに書き込む

#### 主要関数

```javascript
/**
 * 設定シートから設定値を読み込む
 * @returns {Object} settings オブジェクト
 */
function loadSettings()

/**
 * センサデータと不快指数をデータシートに記録する
 * @param {Object} sensorData { temperature, humidity, illuminance, discomfortIndex }
 */
function appendSensorData(sensorData)

/**
 * データシートの最新行（一番下）のセンサデータを返す
 * @returns {Object} 最新のセンサデータ行
 */
function getLatestSensorData()
```

---

### 3.3 AirconModule（エアコン制御モジュール）

#### 責務

- エアコンの現在の電源状態・設定温度を管理する
- 照度・不快指数・経過時間に基づいてエアコン操作を判断する
- Nature API を通じてエアコンを操作する

#### 主要関数

```javascript
/**
 * エアコン制御のメイン判断処理
 * @param {Object} sensorData   取得したセンサデータ（不快指数含む）
 * @param {Object} settings     設定値
 * @param {Object} airconState  現在のエアコン状態
 * @returns {Object}            更新後のエアコン状態
 */
function evaluateAndControlAircon(sensorData, settings, airconState)

/**
 * Nature API でエアコンに操作コマンドを送信する
 * @param {string} appliance_id  エアコンのアプライアンスID
 * @param {number} temperature   設定温度
 * @param {string} mode          運転モード（例: "cool"）
 * @param {string} power         電源状態（"on" | "off"）
 */
function sendAirconCommand(appliance_id, temperature, mode, power)

/**
 * エアコン状態をスプレッドシートに保存する（トリガー間の状態引き継ぎ）
 * @param {Object} airconState
 */
function saveAirconState(airconState)

/**
 * エアコン状態をスプレッドシートから復元する
 * @returns {Object} airconState
 */
function loadAirconState()
```

#### エアコン状態オブジェクト（airconState）

```javascript
{
  isPowerOn: boolean,         // 電源ON/OFF
  currentTemp: number,        // 現在の設定温度
  lastStartedAt: Date | null, // 稼働開始時刻
  lastAdjustedAt: Date | null // 最後に温度調節した時刻
}
```

---

### 3.4 TwitterModule（Twitter 投稿モジュール）

#### 責務

- エアコン操作時のツイートを投稿する
- 毎時 00 分の定期ツイートを投稿する

#### 主要関数

```javascript
/**
 * Twitter にツイートを投稿する
 * @param {string} message ツイート本文
 */
function postTweet(message)

/**
 * エアコン操作時のツイート文面を生成する
 * @param {Object} sensorData  現在のセンサデータ
 * @param {number} setTemp     設定温度
 * @param {string} action      操作内容（例: "起動", "温度変更"）
 * @param {string} template    スプレッドシートから取得した雛形文字列
 * @returns {string}
 */
function buildAirconTweetMessage(sensorData, setTemp, action, template)

/**
 * 定期投稿用のツイート文面を生成する
 * @param {Object} latestData  スプレッドシートの最新行データ
 * @param {string} template    雛形文字列
 * @returns {string}
 */
function buildHourlyTweetMessage(latestData, template)
```

#### ツイート雛形の変数プレースホルダー

| プレースホルダー | 内容 |
|---------------|------|
| `{datetime}` | 現在日時（例: 2025-07-01 14:30） |
| `{temperature}` | 室温（℃） |
| `{humidity}` | 湿度（%） |
| `{discomfort_index}` | 不快指数 |
| `{set_temp}` | エアコン設定温度（℃） |
| `{action}` | エアコン操作内容（起動 / 温度変更） |

## 4. データ設計

### 4.1 スプレッドシート構成

| シート名 | 用途 |
|---------|------|
| `設定` | ユーザー設定値の入力・管理 |
| `データ` | センサデータ・不快指数の時系列記録 |
| `エアコン状態` | トリガー間をまたぐエアコン状態の永続化 |

---

### 4.2 設定シート（`設定`）

| セル | 設定項目 | デフォルト値 | 型 |
|-----|---------|------------|---|
| B2 | 不快指数しきい値（冷房起動） | `75` | number |
| B3 | 照度しきい値（冷房制御を有効化する照度） | `500` | number（lux） |
| B4 | 最低設定温度 | `18` | number（℃） |
| B5 | 最高設定温度（初期起動温度） | `26` | number（℃） |
| B6 | 温度調節間隔（分） | `30` | number |
| B7 | エアコン アプライアンスID | （入力必須） | string |
| B8 | Twitter エアコン操作時 雛形 | （入力必須） | string |
| B9 | Twitter 定期投稿 雛形 | （入力必須） | string |

---

### 4.3 データシート（`データ`）

各行が 1 回の 5 分トリガー実行に対応する。

| 列 | 項目名 | 型 | 説明 |
|----|-------|---|------|
| A | 記録日時 | DateTime | トリガー実行時刻 |
| B | 温度 | number | ℃ |
| C | 湿度 | number | % |
| D | 照度 | number | lux |
| E | 不快指数 | number | スプレッドシート関数で計算（`=0.81*B2+0.01*C2*(0.99*B2-14.3)+46.3`） |
| F | 備考 | string | エラー内容などを任意記録 |

---

### 4.4 エアコン状態シート（`エアコン状態`）

トリガー間の状態を引き継ぐため、以下の値を単一行で永続化する。

| 列 | 項目名 | 型 |
|----|-------|---|
| A | 電源状態 | boolean |
| B | 現在の設定温度 | number |
| C | 稼働開始時刻 | DateTime |
| D | 最終温度調節時刻 | DateTime |

## 5. 処理フロー設計

### 5.1 メイン処理フロー（5 分ごと）

```
START
 │
 ├─ 1. 設定値を読み込む（SheetModule.loadSettings）
 │
 ├─ 2. センサデータ取得（SensorModule.fetchSensorData）
 │     └─ [エラー] → エラーをデータシートに記録して END
 │
 ├─ 3. データシートに記録（SheetModule.appendSensorData）
 │
 ├─ 4. エアコン状態を復元（AirconModule.loadAirconState）
 │
 ├─ 5. エアコン制御判断（AirconModule.evaluateAndControlAircon）
 │     ├─ [照度 ≤ しきい値] → 制御スキップ、状態保存して END
 │     ├─ [電源OFF かつ DI > しきい値] → エアコン起動処理へ
 │     └─ [電源ON かつ 30分経過] → 温度調節処理へ
 │
 ├─ 6. エアコン状態を保存（AirconModule.saveAirconState）
 │
 └─ END
```

### 5.2 エアコン起動処理

```
[照度 > しきい値] かつ [電源OFF] かつ [DI > DI しきい値]
 │
 ├─ 最低設定温度でエアコン起動（sendAirconCommand）
 ├─ airconState.isPowerOn = true、lastStartedAt = 現在時刻
 └─ Twitter にエアコン起動ツイートを投稿（TwitterModule.postTweet）
```

### 5.3 温度調節処理

```
[電源ON] かつ [現在時刻 - lastAdjustedAt >= 30分]
 │
 ├─ [DI > DI しきい値]
 │     └─ currentTemp > 最低設定温度 の場合 → 設定温度 -1℃
 │
 └─ [DI < DI しきい値 - 3]
       └─ currentTemp < 最高設定温度 の場合 → 設定温度 +1℃
 │
 ├─ sendAirconCommand（変更がある場合のみ）
 ├─ lastAdjustedAt = 現在時刻
 └─ Twitter に設定変更ツイートを投稿
```

### 5.4 定期投稿フロー（毎時 00 分）

```
START
 │
 ├─ データシートの最新行を取得（SheetModule.getLatestSensorData）
 ├─ 雛形にデータを代入してツイート文面を生成
 └─ Twitter に投稿（TwitterModule.postTweet）
```

## 6. エアコン制御ロジック設計

### 6.1 制御条件一覧

| 条件 | 処理 | Twitter 投稿 |
|-----|------|------------|
| 照度 ≤ しきい値 | 制御しない | なし |
| 照度 > しきい値 かつ 電源OFF かつ DI > DI しきい値 | 最低設定温度で起動 | あり（起動） |
| 電源ON かつ 30分経過 かつ DI > DI しきい値 かつ 設定温度 > 最低設定温度 | 設定温度 −1℃ | あり（変更） |
| 電源ON かつ 30分経過 かつ DI < DI しきい値 − 3 かつ 設定温度 < 最高設定温度 | 設定温度 +1℃ | あり（変更） |
| 上記以外 | 何もしない | なし |

### 6.2 設定温度の範囲制限

```
最低設定温度 ≤ currentTemp ≤ 最高設定温度
```

- `currentTemp` が最低設定温度のとき → これ以上下げない
- `currentTemp` が最高設定温度のとき → これ以上上げない

### 6.3 不快指数の判定しきい値

| 変数 | 値 | 説明 |
|-----|---|------|
| `DI_threshold` | 設定値（デフォルト 75） | 冷房を起動・強化する基準 |
| `DI_threshold - 3` | `DI_threshold` − 3 | 冷却を緩和（設定温度を上げる）基準 |

## 7. 外部インタフェース設計

### 7.1 Nature API（センサ取得）

| 項目 | 詳細 |
|-----|------|
| エンドポイント | `GET https://api.nature.global/1/devices` |
| 認証 | Bearer トークン（スクリプトプロパティ: `NATURE_API_TOKEN`） |
| 取得対象 | `newest_events.te.val`（温度）、`newest_events.hu.val`（湿度）、`newest_events.il.val`（照度） |

### 7.2 Nature API（エアコン操作）

| 項目 | 詳細 |
|-----|------|
| エンドポイント | `POST https://api.nature.global/1/appliances/{appliance_id}/aircon_settings` |
| 認証 | Bearer トークン（同上） |
| パラメータ | `temperature`, `operation_mode`（例: `cool`）, `button`（`power-on` / `power-off`） |

### 7.3 Twitter API v2

| 項目 | 詳細 |
|-----|------|
| エンドポイント | `POST https://api.twitter.com/2/tweets` |
| 認証 | OAuth 1.0a（スクリプトプロパティ: `TW_API_KEY`, `TW_API_SECRET`, `TW_ACCESS_TOKEN`, `TW_ACCESS_TOKEN_SECRET`） |

## 8. エラーハンドリング設計

### 8.1 エラー処理方針

| エラー種別 | 処理 |
|-----------|------|
| Nature API 呼び出し失敗 | データシートの備考列にエラー内容を記録し、当該トリガー実行を中断 |
| Twitter API 呼び出し失敗 | データシートの備考列にエラー内容を記録し、エアコン制御は継続 |
| スプレッドシート読み書きエラー | Logger にエラーを記録し、実行を中断 |

### 8.2 セキュリティ設計

以下の秘密情報は、すべて Google Apps Script のスクリプトプロパティに安全に保管し、ソースコードに直接記述しない。

| プロパティキー | 内容 |
|-------------|------|
| `NATURE_API_TOKEN` | Nature API アクセストークン |
| `TW_API_KEY` | Twitter API キー |
| `TW_API_SECRET` | Twitter API シークレット |
| `TW_ACCESS_TOKEN` | Twitter アクセストークン |
| `TW_ACCESS_TOKEN_SECRET` | Twitter アクセストークンシークレット |
