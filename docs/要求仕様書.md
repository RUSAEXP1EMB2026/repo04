# 要求仕様書

## 全体概要

### システムの概要

Nature Remo 3 の人感センサー（玄関ドア付近に設置）と Google カレンダー、LINE を連携させ、予定前に「外出動作が確認できていない」場合に段階的な催促通知を行うシステムである。ユーザーは Google カレンダーに予定を登録するだけで、出発タイミングに応じた通知を受け取れる。

### 製品の機能

5 分ごとに Nature API 経由で人感センサーの最終検知時刻を取得し、Google Calendar API で当日の予定と照合する。予定時刻が近いにもかかわらず、直近のドア通過（外出とみなす動き）が確認できない場合、LINE Messaging API で第 1〜第 4 段階の催促通知を送信する。判定結果・通知履歴・設定値は Google Apps Script 内で管理する。

### 想定する利用者の特性

外出予定を忘れがちな人や、出発遅れを防ぎたい人を主な対象とする。

- 予定があるにもかかわらず、自宅でだらだらしてしまい出発が遅れがちな人
- 家族が予定管理し、本人に通知したいケース（例：保護者が子に通知）

## 詳細

### 機能要求

#### 外出確認機能

- システムは、5 分ごとに Nature Remo 3 の人感センサーの最終検知時刻を Nature API（`GET /1/devices`）経由で取得すること
- 人感センサーは玄関ドア前に設置し、予定前後の検知を「外出動作の発生有無」の判定に用いること
- 予定ごとに、開始前監視ウィンドウ内でドア前検知があった場合「外出確認済み」と判定すること
- 判定結果を Google Apps Script のログ基盤に記録すること（判定時刻、センサー最終検知時刻、予定 ID、外出確認状態）

#### 外出予定取得機能

- システムは、Google Calendar API を用いてユーザーが指定したカレンダーから当日の予定を取得すること
- 外出予定の判定は、以下のいずれかの条件を満たす予定を「外出予定」として扱うこと
  - 予定の場所（location）フィールドに値が設定されている
  - 予定のタイトルまたは説明に、ユーザーが Script Properties に登録したキーワード（例：「外出」「出発」「通院」「会議」など）が含まれている
- 終日イベントは外出予定の対象外とすること（ただし、Script Properties の設定で対象に含めるかを変更できること）
- 取得した予定情報（タイトル、開始時刻、場所）を Google Apps Script のログ基盤に記録すること

#### 外出催促通知機能

- 外出予定の開始時刻の 30 分前（デフォルト値、Script Properties で変更可能）から監視を開始すること
- 以下のタイミングで「外出確認済み」でない場合、LINE メッセージを送信すること
  - **第 1 段階（準備リマインド）**：予定開始時刻の 30 分前に未確認の場合
  - **第 2 段階（出発催促）**：予定開始時刻の 10 分前に未確認の場合
  - **第 3 段階（警告）**：予定開始時刻を過ぎても未確認の場合
  - **第 4 段階（繰り返し警告）**：予定開始後、5 分ごとに未確認であれば繰り返し送信（同一予定で最大 6 回）
- 監視中にドア前検知を確認した時点で、当該予定への通知を停止し、「外出を確認しました。いってらっしゃい！」を送信すること
- 各段階のメッセージには、予定のタイトル、開始時刻、現在時刻を含めること

#### 通知制御機能

- ユーザーは、LINE からの返信メッセージ「OK」または「スヌーズ」で一時的に通知を停止できること（スヌーズ時間はデフォルト 15 分、Script Properties で変更可能）
- ユーザーは、LINE からの返信メッセージ「キャンセル」で当該予定に対する通知を完全に停止できること
- 深夜帯（22:00〜7:00、Script Properties で変更可能）は通知を送信しないこと

#### 状態遷移・判定ロジック

- システムは予定ごとに、以下の状態を管理すること
  - `予定あり（未外出） -> 通知中（第 1〜4 段階） -> 外出完了（通知停止）`
  - `予定あり（未外出） -> スヌーズ中 -> 予定あり（未外出）`
- 外出未確認判定は、以下の式を基準とすること
  - `外出未確認 = (現在時刻 - 最終検知時刻) < しきい値` かつ監視対象時間内
- 通知可否フラグは、`予定開始時刻 - 現在時刻` および `スヌーズ終了時刻` に基づき評価すること

#### ログ記録機能

- 外出確認判定結果、予定の照合結果、通知の送信履歴を Google Apps Script の Cloud Logging に記録すること
- ログには以下の情報を含めること
  - 記録日時
  - 人感センサー最終検知時刻
  - 外出確認状態（確認済み/未確認）
  - 照合した予定のタイトル・開始時刻（該当がある場合）
  - 通知の送信有無・段階・メッセージ内容（送信した場合）

#### 設定機能

- ユーザーは、以下の設定項目を Google Apps Script の Script Properties で変更できること
  - 外出予定と判定するキーワードの一覧
  - 終日イベントを外出予定に含めるかどうか（ON/OFF）
  - 外出未確認判定のしきい値（最終検知時刻との差分、デフォルト: 15 分）
  - 第 1 段階の通知タイミング（予定開始何分前か、デフォルト: 30 分前）
  - 第 2 段階の通知タイミング（予定開始何分前か、デフォルト: 10 分前）
  - 第 3 段階以降の繰り返し間隔（デフォルト: 5 分）
  - 同一予定に対する最大通知回数（デフォルト: 6 回）
  - スヌーズ時間（デフォルト: 15 分）
  - 通知を行わない時間帯（デフォルト: 22:00〜7:00）
  - 対象とする Google カレンダーの ID

### 非機能要求

- 外出確認判定および予定照合の処理は 5 分ごとに実行され、処理の遅延は 1 分以内であること
- LINE メッセージの送信は、トリガー条件を満たしてから 1 分以内に行われること
- Nature API の呼び出しはレートリミット（5 回/分）を超えないよう制御すること
- Google Calendar API の呼び出しは、API のクォータ制限を超えないよう制御すること
- LINE Messaging API の無料枠（月 200 通、2024 年時点）を考慮し、過度な通知を抑制する仕組みを備えること
- システムは Google Apps Script 上で動作し、5 分間隔のトリガーで自動実行されること
- 設定管理、状態管理、ログ記録は Google Apps Script 内（Script Properties / Cloud Logging）に集約すること
- Nature API のアクセストークン、LINE Messaging API のチャネルアクセストークン、各種設定値は Script Properties に安全に保管すること
- API 呼び出しのエラー（ネットワークエラー、認証エラー、レートリミット超過など）が発生した場合、エラー内容を Cloud Logging に記録し、次回の実行時に処理を継続すること
- システムの正常動作を確認するため、毎日 8:00 に「システム正常稼働中」のメッセージを LINE に送信すること（Script Properties で ON/OFF 設定可能）

### 制約条件

- Nature Remo 3 の人感センサーは赤外線ベースのモーションセンサーであり、静止人物の検知は困難なため、在宅推定ではなく「ドア前の動き検知による外出確認」に用途を限定する
- Nature API から取得できる情報は「最後に動きを検知した時刻」のみであり、ドア通過そのものを直接識別できない
- Google Apps Script のトリガー実行間隔の最小値は 1 分であるが、実行タイミングには数十秒の誤差が生じうる
- LINE Messaging API の無料プラン（コミュニケーションプラン）では月間メッセージ送信数に上限があるため、通知回数の上限設定が必要である

### 外部インタフェース

| インタフェース | 用途 | 通信方式 |
|---|---|---|
| Nature API（`api.nature.global`） | 人感センサー最終検知時刻の取得 | HTTPS REST API（Bearer トークン認証） |
| Google Calendar API | ユーザーの予定取得 | Google Apps Script 内蔵サービス |
| LINE Messaging API | 通知メッセージの送信・ユーザーからの返信受信 | HTTPS REST API（チャネルアクセストークン認証） |

### システム構成

```
[Nature Remo 3 (玄関ドア前の人感センサー)]
        |
        | Nature API (HTTPS)
        v
[Google Apps Script (Script Properties / Cloud Logging)]
        |        
        | Google Calendar API
        v
   [Google カレンダー (予定取得)]
        |
        | LINE Messaging API (HTTPS)
        v
[LINE (通知送信・返信受信)]
        |
        v
[ユーザー]
```
