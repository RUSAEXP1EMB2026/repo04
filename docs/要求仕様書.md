# 要求仕様書

## 全体概要

### システムの概要

Nature Remo 3 の人感センサーと Google カレンダー、LINE を連携させ、外出予定があるにもかかわらず在宅していることを検知し、LINE で通知（警告）を行うシステムである。ユーザーは Google カレンダーに外出予定を登録しておくだけで、出発すべき時刻を過ぎても自宅にいる場合に LINE メッセージで催促を受けることができる。

### 製品の機能

5 分ごとに Nature Remo 3 の人感センサー（赤外線モーションセンサー）の値を Nature API 経由で取得し、在宅状況を判定する。並行して Google Calendar API を用いてユーザーの予定を取得し、外出が必要な予定があるかを確認する。外出予定の開始時刻が近づいている、または過ぎているにもかかわらず人感センサーが在宅を示している場合、LINE Messaging API を通じて段階的に警告メッセージを送信する。さらに、Google スプレッドシートに在宅状況・予定の照合結果・通知履歴をログとして記録し、ユーザーがシステムの動作状況を確認できるようにする。

### 想定する利用者の特性

外出予定を忘れがちな人や、時間にルーズで出発が遅れがちな人を主な対象とする。特に以下のような人にとって有益なシステムであると考える。

- 予定があるにもかかわらず、自宅でだらだらしてしまい出発が遅れがちな人
- 子供や学生で、塾・習い事・学校行事などの予定に遅刻しがちな人（保護者がカレンダーに予定を登録しておくことで、本人に通知が届く）
- 一人暮らしで、誰も出発を催促してくれる人がいない人
- 共同生活者が、相手の外出予定を管理し、必要に応じて催促したい場合（例：親が子供に対して）

## 詳細

### 機能要求

#### 在宅検知機能

- システムは、5 分ごとに Nature Remo 3 の人感センサーの最終検知時刻を Nature API（`GET /1/devices`）経由で取得すること
- 直近 15 分以内に人感センサーが人の動きを検知した場合、「在宅」と判定すること
- 15 分以上人感センサーの反応がない場合、「不在」と判定すること
- 在宅・不在の判定結果を Google スプレッドシートに記録すること（判定時刻、センサー最終検知時刻、判定結果）

#### 外出予定取得機能

- システムは、Google Calendar API を用いてユーザーが指定したカレンダーから当日の予定を取得すること
- 外出予定の判定は、以下のいずれかの条件を満たす予定を「外出予定」として扱うこと
  - 予定の場所（location）フィールドに値が設定されている
  - 予定のタイトルまたは説明に、ユーザーがスプレッドシートの所定のセルに登録したキーワード（例：「外出」「出発」「通院」「会議」など）が含まれている
- 終日イベントは外出予定の対象外とすること（ただし、ユーザーがスプレッドシートで終日イベントも対象に含める設定を変更できること）
- 取得した予定情報（タイトル、開始時刻、場所）を Google スプレッドシートに記録すること

#### 外出催促通知機能

- 外出予定の開始時刻の 30 分前（デフォルト値、スプレッドシートで変更可能）から在宅状況の監視を開始すること
- 以下のタイミングで在宅が検知された場合、LINE メッセージを送信すること
  - **第 1 段階（準備リマインド）**：予定開始時刻の 30 分前に在宅の場合、穏やかなリマインドメッセージを送信する（例：「📅 30 分後に『〇〇』の予定があります。そろそろ準備を始めましょう！」）
  - **第 2 段階（出発催促）**：予定開始時刻の 10 分前に在宅の場合、出発を促すメッセージを送信する（例：「⚠️ 10 分後に『〇〇』が始まります！まだ家にいるようです。早く出発してください！」）
  - **第 3 段階（警告）**：予定開始時刻を過ぎても在宅の場合、強い警告メッセージを送信する（例：「🚨『〇〇』の時間を過ぎています！！今すぐ出発してください！！！」）
  - **第 4 段階（繰り返し警告）**：予定開始時刻を過ぎた後、5 分ごとに在宅が継続していれば警告メッセージを繰り返し送信すること。ただし、同一予定に対する通知は最大 6 回（第 3 段階以降）までとすること
- 在宅状況が「不在」に変わった時点で、当該予定に対する通知を停止し、「外出を確認しました。いってらっしゃい！」のメッセージを送信すること
- 各段階のメッセージには、予定のタイトル、開始時刻、現在時刻を含めること

#### 通知制御機能

- ユーザーは、LINE からの返信メッセージ「OK」または「スヌーズ」で一時的に通知を停止できること（スヌーズ時間はデフォルト 15 分、スプレッドシートで変更可能）
- ユーザーは、LINE からの返信メッセージ「キャンセル」で当該予定に対する通知を完全に停止できること
- 深夜帯（22:00〜7:00、スプレッドシートで変更可能）は通知を送信しないこと

#### ログ記録機能

- 在宅判定結果、予定の照合結果、通知の送信履歴を Google スプレッドシートに記録すること
- ログには以下の情報を含めること
  - 記録日時
  - 人感センサー最終検知時刻
  - 在宅判定結果（在宅/不在）
  - 照合した予定のタイトル・開始時刻（該当がある場合）
  - 通知の送信有無・段階・メッセージ内容（送信した場合）

#### 設定機能

- ユーザーは、以下の設定項目を Google スプレッドシートの所定のセルに入力することで変更できること
  - 外出予定と判定するキーワードの一覧
  - 終日イベントを外出予定に含めるかどうか（ON/OFF）
  - 在宅判定のしきい値（人感センサーの無反応時間、デフォルト: 15 分）
  - 第 1 段階の通知タイミング（予定開始何分前か、デフォルト: 30 分前）
  - 第 2 段階の通知タイミング（予定開始何分前か、デフォルト: 10 分前）
  - 第 3 段階以降の繰り返し間隔（デフォルト: 5 分）
  - 同一予定に対する最大通知回数（デフォルト: 6 回）
  - スヌーズ時間（デフォルト: 15 分）
  - 通知を行わない時間帯（デフォルト: 22:00〜7:00）
  - 対象とする Google カレンダーの ID

### 非機能要求

- 在宅判定および予定照合の処理は 5 分ごとに実行され、処理の遅延は 1 分以内であること
- LINE メッセージの送信は、トリガー条件を満たしてから 1 分以内に行われること
- Nature API の呼び出しはレートリミット（5 回/分）を超えないよう制御すること
- Google Calendar API および Google Sheets API の呼び出しは、各 API のクォータ制限を超えないよう制御すること
- LINE Messaging API の無料枠（月 200 通、2024 年時点）を考慮し、過度な通知を抑制する仕組みを備えること
- システムは Google Apps Script 上で動作し、5 分間隔のトリガーで自動実行されること
- Nature API のアクセストークン、LINE Messaging API のチャネルアクセストークンは Google Apps Script のスクリプトプロパティに安全に保管すること
- API 呼び出しのエラー（ネットワークエラー、認証エラー、レートリミット超過など）が発生した場合、エラー内容をスプレッドシートのログシートに記録し、次回の実行時に処理を継続すること
- システムの正常動作を確認するため、毎日 8:00 に「システム正常稼働中」のメッセージを LINE に送信すること（スプレッドシートで ON/OFF 設定可能）

### 制約条件

- Nature Remo 3 の人感センサーは赤外線ベースのモーションセンサーであり、静止している人の検知は困難であるため、在宅判定の精度には限界がある
- Nature API から取得できる人感センサーの情報は「最後に人の動きを検知した時刻」のみであり、リアルタイムの在不在を直接取得することはできない
- Google Apps Script のトリガー実行間隔の最小値は 1 分であるが、実行タイミングには数十秒の誤差が生じうる
- LINE Messaging API の無料プラン（コミュニケーションプラン）では月間メッセージ送信数に上限があるため、通知回数の上限設定が必要である

### 外部インタフェース

| インタフェース | 用途 | 通信方式 |
|---|---|---|
| Nature API（`api.nature.global`） | 人感センサーの値取得 | HTTPS REST API（Bearer トークン認証） |
| Google Calendar API | ユーザーの予定取得 | Google Apps Script 内蔵サービス |
| Google Sheets API | 設定値の読み取り・ログの書き込み | Google Apps Script 内蔵サービス |
| LINE Messaging API | 通知メッセージの送信・ユーザーからの返信受信 | HTTPS REST API（チャネルアクセストークン認証） |

### システム構成

```
[Nature Remo 3 (人感センサー)]
        |
        | Nature API (HTTPS)
        v
[Google Apps Script] -----> [Google スプレッドシート (設定・ログ)]
        |         |
        |         | Google Calendar API
        |         v
        |    [Google カレンダー (予定取得)]
        |
        | LINE Messaging API (HTTPS)
        v
[LINE (通知送信・返信受信)]
        |
        v
[ユーザー]
```
